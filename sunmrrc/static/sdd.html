System Architecture

End-to-End System Architecture

Four-layer architecture: Mobile Browser → FastAPI SunMRRC Server → Shared DSP Modules → SunSDR2 DX Hardware. All WebSocket endpoints shown with protocols and data rates.

TX Audio Gain Staging

End-to-End TX Gain Staging · AD-012

Complete TX signal chain from Mic → Client EQ → Opus → WebSocket → Server Hilbert SSB → Drive Gain → tanh Limiter → Device DRIVE. Shows healthy gain values (preamp ×1.5, peak ~0.96 post-tanh, 4% reduction) vs the old distorted path (preamp ×3.0, 75% tanh saturation).

TX Telemetry (0x1F00)

0x1F00 Verified Field Offsets · AD-011

Reverse-engineered from a real 40m drive sweep (expert_40m_drive.pcap, cross-checked against external wattmeter + ExpertSDR3). off30 f32 = forward watts · off16 u16/10 = supply volts · off18 f32 = PA temp °C. The device has NO reverse-power field — cannot compute SWR.

TX EQ4 presets: DEFAULT/MEDIUM/STRONG/RAGCHEW · Compressor 3:1 · Anti-alias 4.5kHz LP
tanh limiterSoft magnitude limit at TX_IQ_PEAK=1.0 — safety net, not primary gain control
Forward power0x1F00 off30 f32 = watts (PEP), direct float, no fit needed
Supply voltage0x1F00 off16 u16/10 = volts (NOT SWR — textbook PSU sag, r=-0.79 with power)
